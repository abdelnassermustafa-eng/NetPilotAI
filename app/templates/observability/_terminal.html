üß™ Troubleshooting & Verification

Read-only diagnostic commands used to verify system health, permissions, event flow, and data ingestion when the UI appears incomplete or out of sync.

Verify Backend API Health

Confirms that the NetPilot backend service is running and responding to requests. 

  curl http://localhost:8000/api/v1/health
Expected output: JSON response indicating service health.
Local-only health check. No AWS interaction.
Verify AWS Caller Identity

Displays the AWS account, user, or role currently in use. Useful for confirming permissions and environment context. 

  aws sts get-caller-identity \
    --output table
Expected output: Account ID, ARN, and user or role name.
Identity inspection does not assume or modify roles.
Verify AWS Region Context

Confirms the default AWS region used by the CLI. Region mismatches are a common cause of missing data. 

  aws configure get region
Expected output: Region name (for example, us-east-1).
Region is resolved via AWS CLI configuration.
Verify Event Flow (EventBridge)

Confirms that events are being emitted and recorded by EventBridge. 

  aws events lookup-events \
    --max-results 5 \
    --output table
Expected output: Recent events with timestamps and sources.
Event availability depends on retention settings.
Verify Log Ingestion

Checks whether logs are actively being written to CloudWatch Logs. 

  aws logs tail /netpilot-ai/backend \
    --since 5m
Expected output: Recent log entries or empty output if idle.
Empty output does not always indicate failure.
Verify Metric Availability

Confirms that metrics exist for a service or resource. Useful when dashboards show no data. 

  aws cloudwatch list-metrics \
    --namespace AWS/EC2 \
    --output table
Expected output: Table of available metric definitions.
Metric publishing may have inherent delay.
Verify SQS Message Backlog

Checks whether messages are accumulating in an SQS queue, which may indicate downstream processing delays. 

  aws sqs get-queue-attributes \
    --queue-url <QUEUE_URL> \
    --attribute-names ApproximateNumberOfMessages \
                      ApproximateNumberOfMessagesNotVisible \
    --output table
Expected output: Approximate visible and in-flight message counts.
Message counts are approximate by design.
üìÑ Logs (CloudWatch Logs)

Commands for inspecting CloudWatch Logs used by NetPilot to display application and service activity.

List Log Groups

Lists all CloudWatch Log Groups in the current region. This verifies that logging destinations exist. 

      aws logs describe-log-groups \
        --output table
Expected output: Table of log group names and retention settings.
Read-only inspection command. No resources are modified.
üìÑ Logs (CloudWatch Logs)

Read-only commands for inspecting CloudWatch Logs. These commands help verify log ingestion, activity, and availability independent of the UI.

List Log Groups

Lists all CloudWatch Log Groups in the current region. Use this to confirm that services are successfully creating log destinations. 

    aws logs describe-log-groups \
      --output table
Expected output: Table of log group names and retention settings.
Read-only inspection command. No resources are modified.
Filter Log Groups by Prefix

Lists log groups matching a specific prefix. Useful for large accounts or verifying naming conventions. 

    aws logs describe-log-groups \
      --log-group-name-prefix /netpilot \
      --output table
Expected output: Filtered table of matching log groups.
Safe, read-only filter operation.
List Log Streams

Displays log streams within a specific log group. Recent timestamps indicate active log ingestion. 

    aws logs describe-log-streams \
      --log-group-name /netpilot-ai/backend \
      --order-by LastEventTime \
      --descending \
      --output table
Expected output: Log stream names with last event timestamps.
No log data is modified or deleted.
Retrieve Recent Log Events

Retrieves recent log entries from a specific log stream. Useful for spot-checking log content. 

    aws logs get-log-events \
      --log-group-name /netpilot-ai/backend \
      --log-stream-name <LOG_STREAM_NAME> \
      --limit 20 \
      --output json
Expected output: JSON array of log events with timestamps and messages.
Replace <LOG_STREAM_NAME> with a stream name from the previous command.
Tail Logs (Near Real-Time)

Streams new log events as they arrive. Best for confirming live backend or service activity. 

    aws logs tail /netpilot-ai/backend \
      --since 10m \
      --follow
Expected output: Continuous log lines as events are written.
Streaming is read-only and does not affect log retention.
Filter Log Events by Pattern

Searches log events for specific keywords or patterns, such as errors or warnings. 

    aws logs filter-log-events \
      --log-group-name /netpilot-ai/backend \
      --filter-pattern "ERROR" \
      --limit 20 \
      --output json
Expected output: JSON array of matching log events.
Filtering does not alter stored log data.
Verify Log Retention Settings

Displays log retention policies for all log groups. Useful for compliance and cost verification. 

    aws logs describe-log-groups \
      --query 'logGroups[*].[logGroupName,retentionInDays]' \
      --output table
Expected output: Log group names with retention periods (if set).
Blank retention values indicate indefinite storage.
üö® Alerts (CloudWatch Alarms)

Read-only commands for inspecting CloudWatch Alarms. These commands help verify alarm existence, state, history, and configuration independent of the UI.

List Alarms

Lists all CloudWatch alarms in the current region. Use this to confirm that alerting rules exist. 

  aws cloudwatch describe-alarms \
    --output table
Expected output: Table of alarm names, states, and metrics.
Read-only inspection command. No alarms are modified.
Filter Alarms by State

Displays alarms in a specific state, such as ALARM or OK. Useful for quickly identifying active issues. 

  aws cloudwatch describe-alarms \
    --state-value ALARM \
    --output table
Expected output: Table of alarms currently in the selected state.
State filtering does not change alarm behavior.
Inspect Alarm Configuration

Displays the full configuration of a specific alarm, including thresholds, metrics, and evaluation periods. 

  aws cloudwatch describe-alarms \
    --alarm-names <ALARM_NAME> \
    --output json
Expected output: JSON description of the alarm configuration.
Replace <ALARM_NAME> with a valid alarm name.
View Alarm History

Retrieves recent state transitions and actions for a specific alarm. 

  aws cloudwatch describe-alarm-history \
    --alarm-name <ALARM_NAME> \
    --history-item-type StateUpdate \
    --max-items 10 \
    --output table
Expected output: Table of recent alarm state changes.
History inspection does not trigger notifications.
List Metric Alarms

Displays only metric-based alarms. Useful when composite alarms are also present. 

  aws cloudwatch describe-alarms \
    --alarm-types MetricAlarm \
    --output table
Expected output: Table of metric alarms.
Composite alarms are excluded from this view.
Verify Alarm Actions

Checks which notification actions are attached to alarms, such as SNS topics. 

  aws cloudwatch describe-alarms \
    --query 'MetricAlarms[*].[AlarmName,AlarmActions]' \
    --output table
Expected output: Alarm names with configured actions.
Action inspection does not invoke notifications.
üìä Metrics (CloudWatch Metrics)

Read-only commands for inspecting CloudWatch Metrics. These commands help verify metric availability, namespaces, dimensions, and data points independent of the UI.

List Metric Namespaces

Lists all available metric namespaces in the current region. Useful for confirming that services are publishing metrics. 

  aws cloudwatch list-metrics \
    --query 'Metrics[*].Namespace' \
    --output text
Expected output: List of metric namespaces (may contain duplicates).
Namespace listing is read-only and does not retrieve metric values.
List Metrics in a Namespace

Displays metrics published under a specific namespace. Helps verify that expected metrics exist. 

  aws cloudwatch list-metrics \
    --namespace AWS/EC2 \
    --output table
Expected output: Table of metric names and dimensions.
Replace AWS/EC2 with the desired namespace.
Filter Metrics by Name

Filters metrics by metric name within a namespace. Useful for identifying a specific signal. 

  aws cloudwatch list-metrics \
    --namespace AWS/EC2 \
    --metric-name CPUUtilization \
    --output table
Expected output: Matching metric definitions with dimensions.
Listing metrics does not retrieve datapoints.
Retrieve Metric Statistics

Retrieves aggregated datapoints for a metric over a time range. Useful for verifying metric values and trends. 

  aws cloudwatch get-metric-statistics \
    --namespace AWS/EC2 \
    --metric-name CPUUtilization \
    --statistics Average \
    --period 300 \
    --start-time 2026-01-15T00:00:00Z \
    --end-time 2026-01-15T01:00:00Z \
    --output json
Expected output: JSON datapoints with timestamps and values.
Time range and dimensions must match published metric data.
Retrieve Metric Data (Multiple Metrics)

Retrieves datapoints for one or more metrics using metric queries. Recommended for dashboards and batch inspection. 

  aws cloudwatch get-metric-data \
    --metric-data-queries file://metric-queries.json \
    --start-time 2026-01-15T00:00:00Z \
    --end-time 2026-01-15T01:00:00Z \
    --output json
Expected output: JSON metric results for each query.
Metric queries are defined externally for clarity and reuse.
Verify Metric Publishing Delay

Checks whether metrics are being published with delay. Useful when UI appears out of sync. 

  aws cloudwatch get-metric-statistics \
    --namespace AWS/EC2 \
    --metric-name CPUUtilization \
    --statistics Maximum \
    --period 300 \
    --start-time 2026-01-15T00:55:00Z \
    --end-time 2026-01-15T01:00:00Z \
    --output json
Expected output: Recent datapoints or empty result if delayed.
CloudWatch metrics may have inherent publishing latency.
üì° Events (EventBridge, SQS, SNS)

Read-only commands for inspecting AWS event flow. These commands help verify EventBridge rules, targets, and downstream messaging (SQS / SNS) independent of the UI.

List EventBridge Rules

Lists all EventBridge rules in the current region. Use this to confirm that event sources and routing rules exist. 

  aws events list-rules \
    --output table
Expected output: Table of rule names, states, and event buses.
Read-only inspection command. No rules are modified.
Filter EventBridge Rules by Prefix

Displays rules matching a specific name prefix. Useful for identifying NetPilot-managed rules. 

  aws events list-rules \
    --name-prefix netpilot \
    --output table
Expected output: Filtered table of matching rules.
Prefix filtering does not affect rule execution.
List EventBridge Rule Targets

Displays targets attached to a specific EventBridge rule, such as SQS queues, SNS topics, or Lambda functions. 

  aws events list-targets-by-rule \
    --rule <RULE_NAME> \
    --output table
Expected output: Table of rule targets and ARNs.
Replace <RULE_NAME> with an existing rule name.
Lookup Recent Events

Retrieves recent events recorded by EventBridge. Useful for confirming that events are being emitted. 

  aws events lookup-events \
    --max-results 10 \
    --output table
Expected output: Table of recent events with timestamps and sources.
Event history availability depends on EventBridge retention.
Receive Messages from SQS Queue

Retrieves messages from an SQS queue used as an event target. This helps verify end-to-end event delivery. 

  aws sqs receive-message \
    --queue-url <QUEUE_URL> \
    --max-number-of-messages 5 \
    --wait-time-seconds 1 \
    --output json
Expected output: JSON array of messages (if available).
Receiving messages does not delete them unless explicitly requested.
Inspect SQS Queue Attributes

Displays message counts and queue state. Useful for diagnosing backlogs or delivery delays. 

  aws sqs get-queue-attributes \
    --queue-url <QUEUE_URL> \
    --attribute-names ApproximateNumberOfMessages \
                      ApproximateNumberOfMessagesNotVisible \
    --output table
Expected output: Approximate message counts.
Message counts are approximate by design.
List SNS Topics

Lists SNS topics used for notifications or event fan-out. Helps verify alerting and messaging configuration. 

  aws sns list-topics \
    --output table
Expected output: Table of SNS topic ARNs.
Listing topics does not publish or modify messages.
üåê Network Inventory (VPC, Subnets, Routing, Gateways)

Read-only commands for inspecting AWS network resources. These commands verify network topology, routing, and connectivity components independent of the UI.

VPCs
List VPCs

Displays all Virtual Private Clouds in the current region. Confirms VPC existence and CIDR allocation. 

  aws ec2 describe-vpcs \
    --query 'Vpcs[*].[VpcId,CidrBlock,IsDefault,State]' \
    --output table
Expected output: Table of VPC IDs, CIDR blocks, and default status.
Read-only inspection. No network resources are modified.
Subnets
List Subnets

Displays all subnets and their availability zones. Useful for verifying subnet layout. 

  aws ec2 describe-subnets \
    --query 'Subnets[*].[SubnetId,VpcId,CidrBlock,AvailabilityZone]' \
    --output table
Expected output: Table of subnets with CIDR blocks and AZs.
Read-only network inspection.
Route Tables
List Route Tables

Displays route tables and their associated VPCs. Confirms routing configuration presence. 

  aws ec2 describe-route-tables \
    --query 'RouteTables[*].[RouteTableId,VpcId]' \
    --output table
Expected output: Table of route table IDs and VPC associations.
Routes are not modified or applied.
Inspect Routes

Displays detailed routes within a specific route table. Useful for validating internet or NAT routing. 

  aws ec2 describe-route-tables \
    --route-table-ids <ROUTE_TABLE_ID> \
    --output json
Expected output: JSON route definitions.
Replace <ROUTE_TABLE_ID> with a valid route table ID.
Internet Gateways
List Internet Gateways

Displays all Internet Gateways and their attachments. Confirms internet access configuration. 

  aws ec2 describe-internet-gateways \
    --query 'InternetGateways[*].[InternetGatewayId,Attachments]' \
    --output table
Expected output: IGW IDs and attached VPCs.
Gateway attachments are not modified.
NAT Gateways
List NAT Gateways

Displays NAT Gateways and their operational state. Useful for verifying outbound internet paths. 

  aws ec2 describe-nat-gateways \
    --query 'NatGateways[*].[NatGatewayId,VpcId,SubnetId,State]' \
    --output table
Expected output: Table of NAT Gateways with state and subnet.
NAT resources are not created or deleted.
Security Groups
List Security Groups

Displays all security groups in the current region along with their VPC associations. Useful for verifying segmentation and isolation boundaries. 

  aws ec2 describe-security-groups \
    --query 'SecurityGroups[*].[GroupId,GroupName,VpcId]' \
    --output table
Expected output: Table of security group IDs, names, and VPC IDs.
Read-only inspection. No rules are modified.
Inspect Security Group Rules

Displays inbound and outbound rules for a specific security group. Useful for validating allowed traffic. 

  aws ec2 describe-security-groups \
    --group-ids <SECURITY_GROUP_ID> \
    --output json
Expected output: JSON description of ingress and egress rules.
Replace <SECURITY_GROUP_ID> with a valid group ID.
Network Interfaces (ENIs)
List Network Interfaces

Displays all Elastic Network Interfaces in the region. Useful for verifying attachments to EC2 instances, NAT gateways, and load balancers. 

  aws ec2 describe-network-interfaces \
    --query 'NetworkInterfaces[*].[NetworkInterfaceId,SubnetId,VpcId,Status]' \
    --output table
Expected output: Table of ENI IDs, subnets, VPCs, and status.
Network interfaces are not attached or detached by this command.
Inspect Network Interface Details

Displays detailed configuration for a specific ENI, including private IPs, attachments, and security groups. 

  aws ec2 describe-network-interfaces \
    --network-interface-ids <ENI_ID> \
    --output json
Expected output: JSON description of the ENI configuration.
Replace <ENI_ID> with a valid network interface ID.
